# 資料詞彙表（修訂版）

適用系統：紀錄 / 通知 / 退款系統（會員、教職員、管理員）  
修訂依據：15 個功能流程圖版本

---

## 一、修訂重點（先看這裡）

### 建議新增
- `users`：統一登入主體（會員 / 教職員 / 管理員），支援「登入成功/失敗」流程。
- `login_logs`：記錄登入結果與失敗原因，對應流程中的「顯示登入錯誤」。
- `notification_preferences`：會員通知偏好（LINE/Gmail/是否啟用）。
- `notification_deliveries`：通知發送結果、重送次數、錯誤訊息。
- `refund_attachments`：退款佐證檔案（對應「上傳佐證資料」）。
- `refund_reviews`：管理員審核紀錄（核准/拒絕、原因、審核時間）。
- `refund_transactions`：退款金流處理紀錄（成功/失敗、交易編號）。

### 建議調整
- 退款狀態建議統一為：`PENDING_REVIEW` / `APPROVED` / `REJECTED` / `PROCESSING` / `REFUNDED` / `FAILED` / `PARTIAL_REFUNDED`。
- `refund_id` 型別由 `INT` 改為 `VARCHAR(20)`（你範例是 `R001`，與 INT 不一致）。
- 通知管道建議不要用 `LINE+Gmail` 當單一值，改以多筆發送紀錄表示（同一通知可有 2 筆 delivery）。
- 出席欄位 `status`、`session_status` 命名建議統一為 `attendance_status`。

### 建議刪減或避免重複
- 同一意義欄位避免分散在多表重複定義（例如 `member_name`、`course_name`）。  
  可保留於交易快照欄位，但主查詢建議由關聯表取得。

## 三、原文件對照（新增 / 修改 / 刪減建議）

### A. 新增欄位（建議）
- 退款：`last_updated_at`、`review_comment`、`gateway_txn_no`、`fail_reason`、`partial_refund` 相關狀態。
- 通知：`retry_count`、`error_message`、通知偏好（`line_enabled`、`gmail_enabled`）。
- 登入：`is_success`、`fail_reason`（登入錯誤追蹤）。

### B. 修改欄位（建議）
- `refund_id` 型態：`INT` -> `VARCHAR(20)`。
- `status`（退款）值集合擴充（加入 `PROCESSING`、`REFUNDED`、`FAILED`、`PARTIAL_REFUNDED`）。
- `channel` 建議改用發送明細一筆一管道，避免 `LINE+Gmail` 字串耦合。

### C. 可刪減或移到快照欄位
- `member_name`、`course_name` 於多處重複；建議主查詢用關聯取得。  
  若需保留歷史語意，可在退款單保留快照欄位：`member_name_snapshot`、`course_name_snapshot`。

## 四、與 15 個流程圖的對應關係（摘要）

- 會員 5 功能：`course_records`、`attendance_records`、`notification_preferences + notification_deliveries`、`refund_requests + refund_attachments`。
- 教職員 4 功能：`teacher_course_assignments`、`courses`、`attendance_records`、`course_records`。
- 管理員 6 功能：`course_records`、`attendance_records`、`notifications + notification_deliveries`、`refund_reviews`、`refund_transactions`、`refund_requests(status 更新)`。
