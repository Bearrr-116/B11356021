# 15 個功能流程圖

以下為依角色拆分的 15 個功能流程圖（Mermaid）。

---

## 1) 會員－查看課程紀錄
```mermaid
flowchart TD
    A[開始] --> B[會員登入系統]
    B --> C{登入成功?}
    C -- 否 --> D[顯示登入錯誤] --> Z[結束]
    C -- 是 --> E[進入會員首頁]
    E --> F[點選 查看課程紀錄]
    F --> G[系統擷取會員ID]
    G --> H[查詢會員已報名/已修課程]
    H --> I[顯示課程紀錄清單]
    I --> Z[結束]
```

## 2) 會員－查看出席與歷史紀錄
```mermaid
flowchart TD
    A[開始] --> B[會員登入系統]
    B --> C{登入成功?}
    C -- 否 --> D[顯示登入錯誤] --> Z[結束]
```

## 3) 會員－接收通知（LINE / Gmail）
```mermaid
flowchart TD
    A[開始] --> B[系統產生通知事件]
    B --> C[讀取會員通知偏好]
    C --> D{通知管道}
    D -- LINE --> E[呼叫LINE API發送]
    D -- Gmail --> F[呼叫Gmail SMTP/API發送]
    E --> G{發送成功?}
    F --> H{發送成功?}
    G -- 否 --> I[記錄失敗並排程重送]
    H -- 否 --> I
    G -- 是 --> J[更新通知紀錄為已送達]
    H -- 是 --> J
    I --> Z[結束]
    J --> Z
```

## 4) 會員－提出退款申請
```mermaid
flowchart TD
    A[開始] --> B[會員登入系統]
    B --> C{登入成功?}
    C -- 否 --> D[顯示登入錯誤] --> Z[結束]
    C -- 是 --> E[點選 提出退款申請]
    E --> F[選擇課程與退款原因]
    F --> G[上傳佐證資料(選填)]
    G --> H[送出申請]
    H --> I[系統驗證資料完整性]
    I --> J{驗證通過?}
    J -- 否 --> K[提示補件/修正資料] --> Z
    J -- 是 --> L[建立退款申請單]
    L --> M[狀態設為 審核中]
    M --> N[通知管理員待審核]
    N --> Z[結束]
```

## 13) 管理員－審核退款申請
```mermaid
flowchart TD
    A[開始] --> B[管理員登入系統]
    B --> C{登入成功?}
    C -- 否 --> D[顯示登入錯誤] --> Z[結束]

    E --> F[查看退款申請清單]

    M --> Z[結束]
```

## 14) 管理員－處理退款流程
```mermaid
flowchart TD
    A[開始] --> B[取得已核准退款案件]
    B --> C[建立退款處理單]
    C --> D[送交金流/財務作業]
    D --> E{退款成功?}
    E -- 否 --> F[記錄失敗原因並重試/轉人工]
    E -- 是 --> G[取得交易編號]
    F --> H[回寫案件狀態]
    G --> H
    H --> Z[結束]
```

## 15) 管理員－更新退款結果
```mermaid
flowchart TD
    A[開始] --> B[管理員登入系統]
    B --> C{登入成功?}
    C -- 否 --> D[顯示登入錯誤] --> Z[結束]
    C -- 是 --> E[開啟退款案件]
    E --> F[填寫退款結果(成功/失敗/部分退款)]
    F --> G[輸入備註與交易資訊]
    G --> H[更新案件狀態]
    H --> I[系統保存異動紀錄]
    I --> J[通知會員最終退款結果]
    J --> Z[結束]
```
