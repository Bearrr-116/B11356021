## 一、帳號管理

---

### 1-1 管理買家帳號

| 項目 | 內容 |
|------|------|
| 使用個案名稱 | 管理買家帳號 |
| 行為者 | 管理員 |
| 目標 | 管理員查詢買家帳號資料，並可修改帳號狀態或停權狀態。 |
| 前提 | 1. 管理員已登入後台系統。<br>2. 管理員具備會員管理權限。 |
| 結束狀態 | 1. 系統儲存帳號狀態異動結果。<br>2. 使用者資料庫完成更新。 |
| Extend 之使用個案 | 停權處理（買家）：當管理員修改帳號狀態為停權時觸發。 |
| Include 之使用個案 | 查詢買家帳號、顯示帳號狀態、修改帳號狀態 |
| Generalization 之使用個案 | 無 |
| 一系列事件 － 基本路徑 | 1. 管理員登入後台系統。<br>2. 管理員進入買家帳號管理頁面。<br>3. 管理員輸入使用者編號查詢買家帳號。<br>4. 系統顯示帳號狀態。<br>5. 管理員檢視帳號資料。<br>6. 管理員修改帳號狀態或停權狀態。<br>7. 系統儲存資料並更新。<br>8. 系統顯示更新結果。 |
| 一系列事件 － 替代路徑 | A1. 查無使用者編號，系統提示「查無此帳號」，返回查詢頁面。 |

### 1-3 停權處理（賣家）

| 項目 | 內容 |
|------|------|
| 使用個案名稱 | 停權處理（賣家） |
| 行為者 | 管理員 |
| 目標 | 管理員對達停權標準之賣家執行停權，限制其交易行為。 |
| 前提 | 1. 管理員已登入後台系統。<br>2. 管理員已選擇欲停權之賣家。<br>3. 需有明確違規依據。 |
| 結束狀態 | 1. 系統更新停權狀態與停權時間至賣家資料庫。<br>2. 系統顯示處理結果。 |
| Extend 之使用個案 | 無 |
| Include 之使用個案 | 顯示違規紀錄與證據、輸入停權原因與日期、更新停權狀態 |
| Generalization 之使用個案 | 無 |
| 一系列事件 － 基本路徑 | 1. 管理員登入後台系統。<br>2. 管理員選擇欲停權的賣家。<br>3. 系統顯示違規紀錄與證據。<br>4. 管理員判斷是否達停權標準。<br>5. 若達標準，管理員輸入停權原因與日期並送出。<br>6. 系統更新資料並顯示結果。 |
| 一系列事件 － 替代路徑 | A1. 管理員判斷未達停權標準，取消操作並返回帳號管理頁面。<br>A2. 永久停權需二次審核，系統提示送出後待上級確認。 |

### 1-4 身分驗證審核（賣家）

| 項目 | 內容 |
|------|------|
| 使用個案名稱 | 身分驗證審核（賣家） |
| 行為者 | 管理員 |
| 目標 | 管理員審核賣家身分驗證資料，設定驗證狀態並記錄驗證時間。 |

| 結束狀態 | 1. 系統記錄驗證時間。<br>2. 系統更新賣家資料庫之驗證狀態（通過／拒絕）。 |
| Extend 之使用個案 | 無 |
| Include 之使用個案 | 顯示待審核賣家資料、比對身分證字號與申請資料、設定驗證狀態、記錄驗證時間 |
| Generalization 之使用個案 | 無 |
| 一系列事件 － 基本路徑 | 1. 管理員登入後台系統。<br>2. 管理員進入身分驗證審核頁面。<br>3. 系統顯示待審核賣家資料。<br>4. 管理員查看身分證字號與相關資料。<br>5. 管理員比對身分證字號與申請資料是否一致。<br>6. 若通過，設定驗證狀態為通過。<br>7. 系統記錄驗證時間並更新資料。 |
| 一系列事件 － 替代路徑 | A1. 若審核不通過，管理員設定驗證狀態為拒絕，系統記錄並更新資料。 |

### 2-1 違規商品下架

| 項目 | 內容 |
|------|------|
| 使用個案名稱 | 違規商品下架 |
| 行為者 | 管理員 |
| 目標 | 管理員對違反平台規範之商品執行強制下架，並保留違規紀錄。 |
| 前提 | 1. 管理員已登入後台系統。<br>2. 管理員已接收檢舉或系統偵測之違規商品資訊。 |
| 結束狀態 | 1. 系統更新商品狀態為下架。<br>2. 商品資料庫完成更新並保留違規紀錄。<br>3. 系統顯示處理結果。 |
| Extend 之使用個案 | 無 |
| Include 之使用個案 | 檢查商品內容、執行商品下架、保留處理紀錄、更新商品狀態 |
| Generalization 之使用個案 | 無 |
| 一系列事件 － 基本路徑 | 1. 管理員接收檢舉或系統偵測的違規商品。<br>2. 管理員檢查商品內容是否違反規範。<br>3. 若違規，管理員將商品下架並保留處理紀錄。<br>4. 系統更新商品狀態。<br>5. 系統顯示處理結果。 |
| 一系列事件 － 替代路徑 | A1. 管理員判斷商品未違規，取消下架操作並關閉案件。 |

### 2-2 異常價格偵測

| 項目 | 內容 |
|------|------|
| 使用個案名稱 | 異常價格偵測 |
| 行為者 | 系統（自動偵測）、管理員 |
| 目標 | 系統定期掃描商品價格，依規則判斷是否異常，並通知管理員進行審查。 |
| 前提 | 1. 商品已完成上架。<br>2. 系統已設定異常價格判斷標準。 |
| 結束狀態 | 1. 異常商品已被系統標記（price_flag 更新為異常）。<br>2. 系統通知管理員進行審查。 |
| Extend 之使用個案 | 惡意價格處理：當管理員收到通知後進入審查流程時觸發。 |
| Include 之使用個案 | 讀取商品價格資料、依規則判斷價格、標記異常商品、通知管理員 |
| Generalization 之使用個案 | 無 |
| 一系列事件 － 基本路徑 | 1. 系統定期掃描所有商品價格。<br>2. 系統讀取商品編號與價格資料。<br>3. 系統依規則判斷價格是否異常。<br>4. 若價格異常，系統標記該商品。<br>5. 系統通知管理員進行審查。 |
| 一系列事件 － 替代路徑 | A1. 若價格正常，系統不進行處理，繼續掃描下一筆商品。 |

### 2-3 惡意價格處理

| 項目 | 內容 |
|------|------|
| 使用個案名稱 | 惡意價格處理 |
| 行為者 | 管理員 |
| 目標 | 管理員審查檢舉資料與價格紀錄，判斷是否為惡意操作，並對成立案件執行商品下架及更新檢舉狀態。 |
| 前提 | 1. 管理員已登入後台系統。<br>2. 存在待審查之惡意價格檢舉資料。<br>3. 需有證據支持。 |
| 結束狀態 | 1. 系統更新檢舉狀態（成立／駁回）。<br>2. 系統記錄處理日期。<br>3. 若成立，商品完成下架。 |
| Extend 之使用個案 | 違規商品下架：當管理員判定惡意價格成立並執行下架時觸發。 |
| Include 之使用個案 | 查看檢舉資料、顯示檢舉狀態與內容、檢查價格紀錄、更新檢舉狀態、記錄處理日期 |
| Generalization 之使用個案 | 無 |
| 一系列事件 － 基本路徑 | 1. 管理員登入後台系統。<br>2. 管理員查看檢舉資料。<br>3. 系統顯示檢舉狀態與內容。<br>4. 管理員檢查價格紀錄。<br>5. 管理員判斷是否為惡意操作。<br>6. 若成立，進行商品下架。<br>7. 系統更新檢舉狀態。<br>8. 系統記錄處理日期。 |
| 一系列事件 － 替代路徑 | A1. 管理員判定不成立，更新檢舉狀態為駁回，系統記錄處理日期後結案。<br>A2. 重大案件可移交檢舉處理或法務流程，系統記錄移交紀錄。 |

### 3-1 檢舉處理（後台）

| 項目 | 內容 |
|------|------|
| 使用個案名稱 | 檢舉處理（後台） |
| 行為者 | 管理員 |
| 目標 | 管理員審查檢舉內容，判斷檢舉是否成立，執行對應處置並通知檢舉人與被檢舉人處理結果。 |
| 前提 | 1. 系統已接收使用者提交之檢舉案件。<br>2. 系統已建立檢舉紀錄並設定待處理狀態。<br>3. 僅管理員可處理檢舉案件。 |
| 結束狀態 | 1. 系統更新檢舉處理狀態與處理結果。<br>2. 系統記錄處理歷程並保留案件資料。<br>3. 系統通知檢舉人與被檢舉人處理結果。 |
| Extend 之使用個案 | 違規商品下架：當檢舉成立且涉及商品違規時觸發。<br>停權處理（賣家）：當檢舉成立且涉及帳號違規時觸發。 |
| Include 之使用個案 | 查詢檢舉內容與相關資料、審查檢舉是否成立、更新檢舉處理狀態、記錄處理歷程、通知檢舉人與被檢舉人 |
| Generalization 之使用個案 | 無 |
| 一系列事件 － 基本路徑 | 1. 系統接收使用者提交之檢舉案件。<br>2. 系統建立檢舉紀錄並設定待處理狀態。<br>3. 管理員查詢檢舉內容、商品資料、聊天紀錄或交易資料。<br>4. 管理員審查檢舉是否成立。<br>5. 若檢舉成立，執行對應處置（下架、警示、停權等）。<br>6. 系統更新檢舉處理狀態與處理結果。<br>7. 系統記錄處理歷程並保留案件資料。<br>8. 系統通知檢舉人與被檢舉人處理結果。 |
| 一系列事件 － 替代路徑 | A1. 若檢舉不成立，管理員直接更新狀態為不成立，系統記錄並通知雙方。 |
