# 白皮書：法條檢索與案例輔助的領域驅動 RAG 系統

## 1. 研究背景與動機
法律知識工作高度依賴可追溯證據。傳統 LLM 雖能生成流暢文字，但在法律場景常出現「條號錯置、法源混淆、案例類比失真」等問題。  
本研究提出領域驅動 RAG 架構，目標是在回答品質與法律可解釋性之間建立可量測平衡。

## 2. 法律知識數位表徵的理論難點

### 2.1 條文結構的層級性
法律文本不是一般平面文章，具有「法規 -> 編章節 -> 條項款目」結構。若切分策略破壞層級，檢索結果會失去法律語境。

### 2.2 語意與法源的雙重約束
同一法律概念可能在不同法規有不同效果；語意相似不等於法效果相同。  
因此檢索必須同時關注「語意相關」與「法源正確」。

### 2.3 判決文本的敘事性
判決書含大量事實背景與程序敘事，核心法律理由分散。若未做爭點摘要化，向量檢索容易被冗長背景噪音干擾。

## 3. 系統創新與技術方案

### 3.1 Hybrid + Parent-Document 的雙層檢索
- BM25 負責條號、法規名、法學術語精準命中
- Dense retrieval 負責事實語意與爭點相似性
- Parent-document 還原法律上下文，讓引用更完整

### 3.2 幻覺抑制機制
- 生成前：cross-encoder reranking 去除低相關候選
- 生成中：grounded prompt 限制只可引用檢索證據
- 生成後：RAGAS 與 TruLens 監控 faithfulness/groundedness，低分答案標記高風險

### 3.3 可解釋輸出
每個答案必須附：
- 法規名稱與條號
- 相關判決摘要識別資訊
- 信心分數與資料覆蓋註記

## 4. 冷啟動與術語對齊策略

### 4.1 冷啟動（Cold Start）
初期資料不足時，以高權威來源建立「核心法條 + 高頻判決摘要」最小知識庫，再逐步增量擴充專門法領域。

### 4.2 專有名詞對齊
建立法律術語 alias dictionary：
- 例如「侵權行為損害賠償」對齊「民法第184條脈絡」
- 例如「不真正連帶」對齊常見實務語彙

查詢時進行術語展開與標準化，減少民間用語與法學用語不一致造成的漏召回。

## 5. 評估設計與預期成果
- 指標：Recall@10、nDCG@10、Faithfulness、Answer Relevancy、Groundedness
- 目標：在維持低延遲下，提升可追溯回答比例並降低無法源依據回答

預期成果為可在法律教育、法務研究與初步案件整理場景中提供可靠輔助，且保留人工專業審查空間。

## 6. 風險與限制
- 系統輸出不應視為正式法律意見
- 判決摘要品質與更新頻率會影響檢索品質
- 法規修正具時效性，需建立版本控管與重建索引流程
